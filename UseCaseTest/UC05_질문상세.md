# UC05. 질문 상세 (Question Detail)

## 화면 설명
오늘의 질문에 답변하거나 수정하는 화면

## 사전 조건
- 로그인 + 그룹 선택 + 오늘의 질문 존재

## UI 요소
- 상단 바: "마음 남기기" 타이틀 + 뒤로가기
- 질문 카드: "오늘의 질문" 라벨 + 질문 텍스트
- 기분 선택: 5개 몽글 캐릭터 (행복/평온/사랑/슬픔/피곤), 라디오 스타일
- 답변 텍스트 입력: 플레이스홀더 "자유롭게 느낌을 적어보세요", 200자 제한 + 카운터
- CTA 버튼: "답변 남기기" (신규) / "답변 수정하기" (수정)
- 가족 답변 섹션 (다른 멤버 답변이 있을 경우)

---

## UseCase

### UC05-1. 새 답변 작성
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | 기분 선택 + 답변 텍스트 입력 후 "답변 남기기" 터치 |
| 정상 흐름 | 기분 선택 → 텍스트 입력 → 제출 → API 호출 → 답변 저장 → 화면 닫힘 → 홈으로 복귀 + 성공 토스트 + 하트 +1 |
| 검증 실패 1 | 기분 미선택 → "오늘의 몽글을 선택해주세요" 팝업 |
| 검증 실패 2 | 텍스트 미입력 → "답변을 입력해주세요" 에러 |
| 실패 흐름 | API 실패 → 에러 토스트 |
| 결과 (성공) | 답변 저장 + 하트 +1 + 홈 화면 갱신 |

### UC05-2. 답변 수정
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | 기존 답변 수정 후 "답변 수정하기" 터치 |
| 정상 흐름 | 수정 → 제출 → 하트 소모 확인 팝업 ("답변을 수정하면 하트 1개가 소모돼요") → 확인 → API 호출 → 답변 업데이트 |
| 실패 흐름 1 | 하트 부족 → 팝업에서 광고 시청 옵션 또는 취소 |
| 실패 흐름 2 | API 실패 → 에러 토스트 |
| 결과 (성공) | 답변 수정 완료 + 하트 -1 |

### UC05-3. 다른 멤버 답변 확인
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | 가족 답변 섹션에서 멤버 답변 카드 확인 |
| 전제 조건 | 본인 답변 완료 상태 |
| 정상 흐름 | 답변 카드 표시 (멤버 이름 + 기분 캐릭터 + 답변 텍스트) |
| 결과 | 가족 멤버의 답변 확인 |
