# UC09. 알림 (Notification)

## 화면 설명
인앱 알림 목록 화면

## 사전 조건
- 로그인 완료

## UI 요소
- 상단 바: "알림" + 뒤로가기 + "모두 읽음" / "모두 제거" 버튼 (알림 존재 시)
- 알림 리스트: 날짜별 섹션, 각 알림 카드 (타입 아이콘, 제목 (미읽음 시 볼드), 상대 시간, 미읽음 점)
- 로딩 상태: 스피너 + "알림을 불러오는 중..."
- 빈 상태: 벨 아이콘 + "알림이 없어요" + "새로운 소식이 오면 알려드릴게요"

## 알림 타입
- NEW_QUESTION: 새 질문 도착
- MEMBER_ANSWERED: 멤버가 답변함
- ALL_ANSWERED: 모든 멤버 답변 완료
- ANSWER_REQUEST: 답변 재촉
- BADGE: 뱃지 획득

## UseCase

### UC09-1. 알림 목록 조회
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | 알림 화면 진입 |
| 정상 흐름 | 화면 진입 → API 호출 → 알림 목록 표시 (날짜별 그룹핑) |
| 빈 데이터 | 알림 없음 → 빈 상태 표시 |
| 실패 흐름 | API 실패 → 에러 토스트 |
| 결과 | 알림 목록 표시 |

### UC09-2. 알림 터치 (읽음 처리)
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | 개별 알림 카드 터치 |
| 정상 흐름 | 터치 → 읽음 처리 API → 관련 화면으로 네비게이션 |
| 결과 | 알림 읽음 + 해당 컨텐츠로 이동 |

### UC09-3. 모두 읽음 처리
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | "모두 읽음" 버튼 터치 |
| 정상 흐름 | 터치 → 전체 읽음 처리 API → 모든 알림 읽음 상태로 변경 |
| 결과 | 전체 알림 읽음 처리 |

### UC09-4. 모두 제거
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | "모두 제거" 버튼 터치 |
| 정상 흐름 | 터치 → 전체 삭제 API → 빈 상태 표시 |
| 결과 | 전체 알림 삭제 |

### UC09-5. 개별 알림 삭제 (스와이프)
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | 알림 카드 스와이프 (iOS) |
| 정상 흐름 | 좌측 스와이프 → 삭제 옵션 → 삭제 |
| 결과 | 해당 알림 삭제 |
