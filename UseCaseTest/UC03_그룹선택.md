# UC03. 그룹 선택/생성/참여 (Group Select)

## 화면 설명
그룹(가족/친구 공간) 선택, 생성, 초대코드 참여를 관리하는 다단계 화면

## 사전 조건
- 로그인 완료 상태

## UI 요소
### 선택 화면
- 헤더: "몽글" + 알림 벨 아이콘 (읽지 않은 알림 시 뱃지)
- 그룹 카드 리스트 (캐릭터 아바타, 멤버 수, 초대 코드)
- "새 공간 만들기" 카드 → 바텀 시트 (새 공간 만들기 / 초대코드로 참여하기)

### 그룹 생성 화면
- 뒤로가기, 그룹 이름 입력 (최대 10자), 닉네임 입력 (최대 10자), 몽글 색상 선택 (5색)
- "만들기" 버튼

### 생성 완료 화면
- 초대 코드 표시, "복사"/"공유" 버튼, "시작하기" 버튼

### 초대코드 참여 화면
- 뒤로가기, 초대 코드 입력 (최대 20자), 닉네임 입력, 색상 선택
- "참여하기" 버튼

---

## UseCase

### UC03-1. 기존 그룹 선택
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | 그룹 카드 터치 |
| 정상 흐름 | 그룹 카드 터치 → 해당 그룹 활성화 API 호출 → 홈 데이터 로드 → **홈 화면으로 이동** |
| 실패 흐름 | API 실패 → 에러 토스트 |
| 결과 | 선택된 그룹의 홈 화면 진입 |

### UC03-2. 새 그룹 만들기
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | "새 공간 만들기" 선택 |
| 정상 흐름 | 그룹 이름 + 닉네임 + 색상 입력 → "만들기" 터치 → API 호출 → **생성 완료 화면** (초대 코드 표시) |
| 검증 실패 1 | 그룹 이름 미입력 → "공간 이름을 입력해주세요" 토스트 |
| 검증 실패 2 | 닉네임 미입력 → "닉네임을 입력해주세요" 토스트 |
| 검증 실패 3 | 그룹 3개 초과 → "그룹은 최대 3개까지 참여할 수 있어요" 팝업 |
| 실패 흐름 | API 실패 → 에러 토스트 |
| 결과 (성공) | 초대 코드 표시 + 공유 가능 |

### UC03-3. 초대코드로 참여
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | "초대코드로 참여하기" 선택 |
| 정상 흐름 | 초대 코드 + 닉네임 + 색상 입력 → "참여하기" 터치 → API 호출 → **홈 화면으로 이동** |
| 검증 실패 1 | 초대 코드 미입력 → "초대 코드를 입력해주세요" 토스트 |
| 검증 실패 2 | 닉네임 미입력 → "닉네임을 입력해주세요" 토스트 |
| 실패 흐름 1 | 유효하지 않은 코드 → "유효하지 않은 초대 코드예요" 토스트 |
| 실패 흐름 2 | 이미 참여한 그룹 → "이미 참여한 그룹이에요" 토스트 |
| 실패 흐름 3 | 그룹 3개 초과 → 최대 그룹 수 팝업 |
| 결과 (성공) | 해당 그룹의 홈 화면 진입 |

### UC03-4. 초대 코드 복사/공유
| 항목 | 내용 |
|------|------|
| 액터 | 사용자 |
| 트리거 | 생성 완료 화면에서 "복사" 또는 "공유" 버튼 터치 |
| 정상 흐름 (복사) | 터치 → 클립보드에 복사 → "초대 코드가 복사되었어요" 토스트 |
| 정상 흐름 (공유) | 터치 → 시스템 공유 시트 표시 |
| 결과 | 초대 코드 전달 완료 |

### UC03-5. 딥링크로 그룹 참여
| 항목 | 내용 |
|------|------|
| 액터 | 외부 사용자 |
| 트리거 | `monggle://join/{code}` 또는 `monggle.app/join/{code}` 링크 클릭 |
| 정상 흐름 | 링크 클릭 → 앱 실행 → 초대 코드 자동 입력 → 참여 화면 표시 |
| 결과 | 초대코드 참여 화면 자동 진입 |
